================================================================================
                  ANONYMIZER – Version History
            MQXLIFF/TMX Document Anonymization Tool
================================================================================


VERSION 1.0 – First functional release
--------------------------------------
- Creation of the application with a web interface for uploading MQXLIFF files.
- Basic anonymization using three layers:
    · Detection of clinical trial IDs (NCT, EudraCT).
    · Detection of personal information (emails, URLs) with Presidio.
    · Detection of biomedical entities (drugs, diseases)
      using a biomedical language model (SciSpaCy).
- Single-file processing.
- Change preview (before/after) prior to download.
- Preservation of the MQXLIFF XML structure, including internal tags
  (<ph/>, <g>) required by memoQ for reimport.


VERSION 2.0 – Detection improvements and intelligent filtering
---------------------------------------------------------------
- Expanded detection of clinical ID formats:
    · ISRCTN, Protocol IDs (ABC-123-XYZ), Subject IDs, Site IDs,
      Lot Numbers, Site Codes, Ethics Committees, Age ranges.
- Detection of pharmaceutical companies (Top 50): Pfizer, Roche,
  Novartis, AstraZeneca, Sanofi, etc.
- Detection of CROs (Top 25): IQVIA, ICON, Syneos, Parexel, etc.
- Detection of blockbuster drugs: Keytruda, Ozempic, Dupixent, etc.
- Detection of clinical platforms: Veeva, Medidata, Oracle Clinical.
- Detection of central laboratories: Q2 Solutions, Labcorp, etc.
- Detection of investigator names, trial dates, and international addresses.
- Detection of hospitals, medical centers, countries, and nationalities.
- Detection of study names, randomization codes, and medical devices.
- Detection of phone numbers, emails, postal codes, and birth dates
  in multiple formats.
- Detection of URLs from international clinical trial registries.
- Intelligent filtering system to avoid over-anonymization:
    · Word-type filter: only anonymizes nouns and proper nouns,
      not verbs or common adjectives.
    · Structured ID detection: requires specific patterns
      (letters + numbers/hyphens).
    · Density brake: if more than 35% of a segment would be anonymized,
      switches to conservative mode.
    · Lowercase validation: if a term appears in lowercase elsewhere
      in the document, it is not anonymized.
- Batch processing: upload and process multiple MQXLIFF files at once,
  with ZIP download.


VERSION 3.0 – Custom dictionary and internal package
---------------------------------------------------
- Custom dictionary: allows uploading a .txt file with
  project-specific sensitive terms (product names, brands, etc.)
  to be anonymized automatically.
    · Accepts one term per line or comma-separated terms.
    · Supports terms with special characters (®, ©, accents).
- Detection of names with registered trademarks (™) and copyright (©).
- Detection of 200+ technology, software, and hardware companies:
    · Microsoft, Oracle, SAP, Salesforce, AWS, Google, Amazon,
      Apple, Dell, Lenovo, Samsung, Sony, Meta, Tesla, Netflix, etc.
- Automatic detection of capitalized product names
  (commercial proper names).
- Search bar in the preview to quickly find specific segments.
- Creation of the internal Windows package:
    · Automatic dependency installer (Dependencies.bat).
    · Application launcher (Anonymizer.bat).
    · Fully functional internal after installation.


VERSION 4.0 – Bilingual processing and TM exclusion
---------------------------------------------------
- Full bilingual processing EN→ES:
    · Presidio processes both English and Spanish segments.
    · Terms detected in the source (EN) are automatically
      propagated to the target (ES) and vice versa.
- Translation Memory (TM) exclusion system:
    · If a segment has more than 20% anonymized text,
      it can be excluded from the translation memory.
    · Prevents heavily modified segments from being imported into memoQ TM.
    · Visual indicators: badges with anonymization percentage
      and dotted red borders on excluded segments.
    · Option to exclude/include each segment individually.
- Term cache system for consistent anonymization:
    · Once a name or term is detected, it is applied
      consistently throughout the document.
    · Case-insensitive comparison.
    · Longer terms are applied first to avoid partial replacements.
- Cleanup of previous version history in the MQXLIFF
  to prevent unanonymized data in the internal history.
- Preservation of memoQ RowHistory elements.
- Detection of Spanish NIF, CIF, DNI, and NIE.


VERSION 5.0 – English interface and precision improvements
----------------------------------------------------------
- Entire user interface translated into English.
- Updated subtitle: "Anonymize bilingual memoQ documents (EN > ES)".
- Improved detection of short segments: now anonymized correctly
  instead of being ignored.
- Detection of hyphenated study codes (e.g., ABC-1234-XYZ).
- Improved preservation of inline tags when anonymized text
  changes length compared to the original.
- Improved phone number detection and refined logic
  to avoid anonymizing entities unnecessarily.
- Complete user manual included in the package.


VERSION 6.0 – Symmetric anonymization, new detections, and .exe
----------------------------------------------------------------
- Fixed asymmetric anonymization between source and target:
    · Previously, some terms (email brands, medical device companies)
      were detected in Spanish but not in English or vice versa.
    · Now detected consistently in both languages.
- Second-pass system: after processing all segments,
  an additional review ensures that terms discovered
  during target processing are also applied to the source,
  and vice versa.
- Fixed the "Not anonymized" indicator:
  now updates correctly when toggling the "Skip" option
  in the preview.
- New detection: Spanish DNI/NIF with hyphen format (e.g., 45678912-X).
- New detection: License plates (Spanish current and legacy formats).
- New detection: Long numeric identifiers (6+ digits: phone numbers,
  SSN, patient IDs, history numbers).
- Short segment exclusion filter: excludes short anonymized segments
  from the output to reduce noise (configurable word count).
- TM exclusion threshold now configurable (default 50%).
- Both source and target are always processed (no manual selection).
- Protected terms whitelist: allows uploading a list of terms
  that should never be anonymized, even if detected by other layers.
- Created an .exe version of the program for one-click installation
  and 100% internal setup with all packages pre-downloaded.
  No Python installation required.


VERSION 6.1 – TMX support
--------------------------
- Added full TMX file format support:
    · Upload and process .tmx files alongside .mqxliff files.
    · TMX translation units (tu/tuv/seg) are parsed and anonymized
      using the same 4-layer pipeline (Regex, Presidio, ScispaCy, Dictionary).
    · Language detection via xml:lang attributes on tuv elements.
    · All features work with TMX: preview, skip segments, TM exclusion,
      short segment filtering, Excel report, batch processing.
- Improved "Baseline" detection: now requires clinical context
  (e.g., "Baseline Visit", "At Baseline") instead of matching standalone.
- Improved alphanumeric code detection for IDs like 715004599ESIVD.
- Fixed Presidio false positives caused by inline XML placeholders.


VERSION 6.2 – QA Check tab (xbench-style)
------------------------------------------
- New QA Check tab inside the same app, independent of the Anonymizer
  pipeline. Validates any TMX or MQXLIFF file with deterministic checks.
- 17 checks across 7 groups:
    · 11 fully deterministic (zero false positives, on by default):
      Empty target, Forbidden terms, Glossary violation,
      Bracket/quote balance, Tag count mismatch, Broken/malformed tags,
      URLs, Emails, Placeholders, Double spaces (auto-fix),
      Leading/trailing whitespace (auto-fix).
    · 6 xbench-style classics (high signal, may produce rare false
      positives, on by default with per-check toggle):
      Untranslated segment, Invisible/control characters (auto-fix),
      Mixed scripts in same word, Numbers differ, Repeated punctuation,
      Final punctuation differs.
- Inline target editing per finding with ↺ Reset; manual edits are
  preserved in the downloaded cleaned file.
- "Use anonymized output" button to QA the file you just produced in
  the Anonymizer tab without re-uploading.
- Glossary support (TXT tab-separated or CSV with source,target columns).
- Forbidden-terms list support (TXT, one term per line).
- HTML report export with EDITED badges on manually edited targets.
- Cleaned file download preserves XML structure, attributes, inline
  tags outside edited segments, and the canonical IDs added by the
  Anonymizer (<prop type="x-document">…</prop>).
- Removed the legacy Hunspell/spylls spell-check subsystem and all
  heuristic / locale-dependent checks (capitalization, partial
  translation, currency/date reformatting, language-specific
  punctuation rules, global consistency, repeated words, tag order)
  to eliminate false positives. Bundled dictionaries removed from the
  source tree (~11 MB cleanup).
- Reintroduced Hunspell-based spell-check as a NEW opt-in QA category
  (Spelling group) — off by default, controlled per-run by the user:
    · 18 total checks now (17 default + 1 opt-in spell-check).
    · Pure-Python via spylls (no compilation; works on Windows OOTB).
    · 5 default languages bundled with the .exe for fully internal use:
      English (en_US), Spanish (es_ES), French (fr_FR),
      German (de_DE), Italian (it_IT) — using the LibreOffice
      dictionaries (~5–8 MB total inside dictionaries/).
    · Auto-detects target language from the file; ISO/IETF codes are
      normalised (e.g. es-MX → es_ES). Unsupported targets are silently
      skipped with a top-of-results notice; the rest of the QA run is
      unaffected.
    · Misspellings highlighted with a red wavy underline (Word /
      LibreOffice style). Per-segment cap of 20 with "… and N more"
      suffix on longer segments.
    · Per-run ignore-words textarea (one per line, case-insensitive)
      for proper nouns, brand names, study acronyms.
    · No auto-fix offered for spelling — picking the right correction
      for a TM entry is a human judgement.
